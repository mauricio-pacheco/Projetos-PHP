I. Introdução
-------------

O que é?!?
----------
Esta é uma versão do PHP-Nuke feita para os usuários que procuram um pouco mais do PHP-Nuke original. Esta
versão vem com todas as correções dos erros descobertos pelo chatserv (o usuário que mais coleciona, organiza e
fornece correções para PHP-Nuke) da NukeResources -http://nukeresources.com-, possui Módulos da NSN (Nuke
Scripts Network) -http://nukescripts.net-, de Bob Marion, possui a prata da casa, o CNB Your_Account na versão
4.4, de André Escudero, além do popular My_eGallery (2.8), Event_Calendar (adaptado para a versão 7.5 por
XPK_Fenix).

O que há nesta versão?!?
------------------------
Esta versão do PHP-Nuke deve ser instalada a partir do zero, ou seja, é ideal que se instale somente para novos
portais.

Nesta versão, como dito anteriormente, você poderá usufruir do que há de melhor para PHP-Nuke atualmente.

Além de vários Módulos para controlar melhor o gerenciamento de seu conteúdo, também nos importamos (e muito) 
com o quesito segurança, por isso é que decidimos instalar o NSN Sentinel, que tem garantido a segurança de muitos e muitos 
portais rodando PHP-Nuke.

II. Instalação
--------------
Para esta versão foi desenvolvida uma nova versão do Instalador Web, desenvolvida por XPK_FENIX, que, além
de implementar as funcionalidades do sistema, agora possui suporte multi-Temas.

O Processo de instalação é simples e rápido, devendo apenas seguir os passos:

1. Obter o pacote CNB-Kayapó.tar.gz
2. Descompactar o pacote;
3. Subir oconteúdo do diretório html para o servidor
4. Acessar: http://www.seusite.com.br/install
5. Seguir os passos de instalação, preenchendo com atenção os campos solicitados.

-=- Com isso, seu PHP-Nuke já estará instalado! -=- 

Como existem diversos Módulos adicionais instalados, é necessário fazer alguns ajustes para que eles fuincionem bem,
por isso, criamos esta lista de Módulos que necessitam de alterações e instruções de como proceder:

Configuração dos Módulos já instalados nesta versão
---------------------------------------------------
>> My_eGallery
É necessário alterar a configuração CHMOD para ambientes (*NIX) nos seguintes arquivos:

1. Sete CHMOD 777 todos os diretórios (categorias) e sub-diretórios (sub-categorias), localizadas em 
/modules/My_eGallery/gallery/ (CHMOD apenas os DIRETÓRIOS, NÃO OS ARQUIVOS!!!
2. 'CHMOD 777' modules/My_eGallery/temp para permitir que usuários enviem imagens. 
Por questões de segurança você deve setar este diretório em outro lugar. O novo caminho pode ser configurado no 
Menu de Administração.
3. 'CHMOD 666' config.php localizado em 'admin/modules/gallery/'
4. O Modulo já está ativo por padrão!
5. Tudo pronto para montar suas galerias. @:)

>> NSN Banners
A maioria das modificações já foram feitas.

Agora, o que é preciso ser explicado é como funcionam os banners e como fazer novos Temas mostrar estes banners:

- No arquivo theme.php de seu Tema preferido (VOCÊ DEVE EDITAR TODOS OS TEMAS QUE FOR UTILIZAR)
   procure pelas funções themeheader() e themefooter(). Substitua o primeiro exemplo pelo
   segundo exemplo:
   (Em alguns Temas antigos você encontrará uma cópia completa do arquivo banners.php, nesse caso
   code, in substitua a SEÇÃO COMPLETAMENTE).

   Arquivos Original themeheader

    if ($banners == 1) {
	include("banners.php");
    }

   NOVAS LINHAS para a função themeheader

    if ($banners == 1) {
        include("includes/banners1.php");
    }

   NOVAS LINHAS para a função themefooter (insira, abaixo da declaração global de variáveis a variável
   $banners to à linhaglobal)

    if ($banners == 1) {
        include("includes/banners2.php");
    }

Você terá que adicionar as seguintes variáveis em seu arquivo theme.php (se você utiliza um Tema que tenha
    arquivos html, você deve editar os seguintes arquivos:

1) VARIÁVEL $tbanners1 MOSTRA OS BANNERS NO CABEÇALHO (header.html)
2) VARIÁVEL $tbanners2 MOSTRA OS BANNERS NO RODAPÉ (footer.html).

Simplificando, basta colocar a variável $tbanners1 onde se queira mostrar os banners no cabeçalho e 
    $tbanners2 onde quiser mostrar os banners no rodapé.


>> NSN Sentinel
Existem várias maneiras e métodos de se configurar o Sentinel, mas vamos relatar o básico que é ter
o Menu de Administração restrito por um login e uma SENHA.

PARA A SEGURANÇA BÁSICA DE SEU PORTAL, É MUITO IMPORTANTE QUE SEJAM SEGUIDOS OS PASSOS AQUI DESCRITOS!!!

1. Entre em http://seusite.com.br/admin.php?op=ABAuthList e adicione o login e a senha que (podem ser diferentes)
cada administrador terá. É EXTREMAMENTE IMPORTANTE que isso seja feito antes de adicionar o arquivo .htaccess
no sistema, o que impossibilitará o login!

2. Entre na Administração do  Sentinel e coloque o caminho do arquivo .htaccess
Geralmente, o caminho é mostrado abaixo do campo.

3. Ative a autenticação por HTTPAccess selecionando:
Autenticação do Administrador: SELECIONE: Autenticação HTTP dos Administradores

4. Sete CHMOD 666 noarquivo .htaccess
Caso não esteja vendo o arquivo via FTP, habilite o filtro do seu cliente de FTP favorito para listar com
a flag -a (o equivalente a um comando shell ls -a) para que sejammostrados arquivos com um . (ocultos).

Caso você sinta que algo deu errado e aparecer a caixa de verificação desenha e mesmo entrando o login e a
senha corretos o sistema não o logar, CALMA.

Entre em seu Banco de Dados e procure a tabela $prefix_snsst_config (onde $prefix é o prefixo utilizado para
as tabelas) e procure o campo http_auth (deve estar em 1) e altere o valor para 0.

Com isso você poderá se logar novamente como administrador e tentar novamente.

>> 2. Porque Kayapó?

O nome Kayapó (homônimos Kaiapó, Caiapó e Cayapó), significa "aquels que se parecem com macacos" e o nome foi
dado pelos próprios integrantes da tribo.

Estes índios habitam o platô central do país (Sul do PA, Norte do MS), entre 300 e 400 metros acima do nível do mar.

Escolhemos este nome inspirados em um povo bastante evoluído social e politicamente, onde não existem chefes,
de todas as tribos. Todas as tribos sabem como devem se portar e os líderes locais são eleitos por merecimento, 
após muita discussão e aprendizado fornecido pelos membros mais experientes da tribo.

O tema indígena surgiu, inclusive, na tentativa de resgatarmos esta fabulosa cultura que está morrendo. Nosso país 
não consegue lutar contra isso, mesmo com os esforços do Governo Federal, Estaduais e a FUNAI, a modernidade está
dentro das tribos e isso não é nada bom.
